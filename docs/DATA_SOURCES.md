# Data Sources

Last updated: 2026-07-02

## Election Results

Official Knesset election results package:

https://data.gov.il/api/3/action/package_show?id=26f9fa06-fcd7-4173-8df5-65797b63e857

Useful notes:

- Knesset 19-25 include both locality-level and ballot-level resources.
- Knesset 16-18 include ballot-level resources; locality totals can be generated by aggregation.
- Party columns are election-specific ballot letters and should not be treated as stable party IDs across elections.

25th Knesset official results site:

https://votes25.bechirot.gov.il/

## Statistical Areas

Official CBS statistical-area polygon layer from the 2008 census:

https://data.gov.il/api/3/action/package_show?id=statistical-area-2008

Resource URL:

https://e.data.gov.il/dataset/6b729165-dc9c-49b3-afc6-eceabd8fef70/resource/3b88cb58-62cd-43ea-9156-7b81ead557b1/download/50400-2008.7z

Caveat:

The source metadata says this layer is for statistical publication and is not an authoritative reconstruction of settlement or other legal boundaries.

## Polling Places

Official polling-place search result:

https://data.gov.il/api/3/action/package_search?q=%D7%A7%D7%9C%D7%A4%D7%99

Observed table fields include locality code, kalpi code, street, house number, polling-place description, regional committee, and district.

Caveats:

- No coordinates.
- No polygons.
- Row count does not match Knesset 25 ballot-result rows, so it should not be assumed to cover 2022.

The generic-table match uses locality code plus normalized kalpi number. These are match-rate diagnostics, not proof that every matched address is historically exact for that election.

## Preferred Export Format For Newly Found Polygon Layers

If an official source offers multiple export formats, prefer:

1. GeoJSON for web-map development.
2. Shapefile as a backup/archive.
3. FGDB only if it contains multiple layers or richer source metadata.
4. KML only for inspection.
5. CSV/JSON only if geometry is explicitly included.

Prefer WGS84 / EPSG:4326 when available.

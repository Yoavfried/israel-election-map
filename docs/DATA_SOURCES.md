Last updated: 2026-07-05

## Election Results

Official Knesset election results package:

https://data.gov.il/api/3/action/package_show?id=26f9fa06-fcd7-4173-8df5-65797b63e857

Useful notes:

- Knesset 19-25 include both locality-level and ballot-level resources.
- Knesset 16-18 include ballot-level resources; locality totals can be generated by aggregation.
- Party columns are election-specific ballot letters and should not be treated as stable party IDs across elections.
- Direct file downloads from some `e.data.gov.il` URLs may be blocked by browser/security interstitials; the datastore API was usable for current investigation.

25th Knesset official results site:

https://votes25.bechirot.gov.il/

## Statistical Areas

Current project raw layer:

- `data/raw/statistical-areas-2022.geojson`
- 1,776 GeoJSON features.
- Useful properties observed: `SEMEL_YISHUV`, `STAT_2022`, `YISHUV_STAT_2022`.
- Locality coverage in this layer: 407 localities.
- 364 localities have exactly one `STAT_2022`.
- 43 localities have multiple `STAT_2022` values.

This is currently the preferred layer for statistical-area mode because it is the 2022 statistical-area geometry the project already has locally.

Older official CBS statistical-area polygon layer from the 2008 census:

https://data.gov.il/api/3/action/package_show?id=statistical-area-2008

Resource URL:

https://e.data.gov.il/dataset/6b729165-dc9c-49b3-afc6-eceabd8fef70/resource/3b88cb58-62cd-43ea-9156-7b81ead557b1/download/50400-2008.7z

Caveat:

The source metadata says this layer is for statistical publication and is not an authoritative reconstruction of settlement or other legal boundaries.

### Address Coverage

Coverage is measured against official ballot-result rows by locality code plus normalized kalpi number. For split result rows such as `3.1`, a base-kalpi match to `3` is accepted. For the generic table, aliases such as `10 -> 1` are also accepted because the generic kalpi codes use a different numeric convention.

Important caveats:

- K22-K25 rows missing polling-place addresses have zero eligible voters, but nonzero actual votes. These are likely special or external ballot rows. They must be retained as unassigned votes instead of dropped.
- K21 election-specific polling-place files were found as historical-looking URLs, but live and archived downloads did not yield usable spreadsheets during this investigation. K21 currently falls back to the generic table.
- Generic-table matches are not proof that every address is historically exact for that election.
- Address-to-statistical-area assignment remains an approximation: it maps a polling-place building to a statistical area, not each voter's home.
- For localities with exactly one 2022 statistical area, ordinary rows without a direct address can still be assigned by locality. This is separate from direct polling-place address assignment and should be stored as separate provenance.

## K23 Statistical Area Field

The K23 polling-place report includes an `AGS`-style statistical-area field, but it does not align well with the 2022 statistical-area GeoJSON.

Observed match against the local 2022 layer:

- K23 rows with AGS: 8,031.
- Unique K23 locality+AGS pairs: 1,570.
- Unique pairs matching 2022 `SEMEL_YISHUV` + `STAT_2022`: 589.
- Unique-pair match rate: 37.5%.

Decision:

Keep K23 AGS as source metadata only. Do not join it directly to the 2022 statistical-area polygons.

## Preferred Export Format For Newly Found Polygon Layers

If an official source offers multiple export formats, prefer:

1. GeoJSON for web-map development.
2. Shapefile as a backup/archive.
3. FGDB only if it contains multiple layers or richer source metadata.
4. KML only for inspection.
5. CSV/JSON only if geometry is explicitly included.

Prefer WGS84 / EPSG:4326 when available.
